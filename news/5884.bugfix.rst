Shorthand marker keys in Pipfile entries (e.g. ``sys_platform = "== 'linux'"``, ``platform_machine = "== 'arm64'"``) now behave identically to the equivalent ``markers`` syntax.  Previously, using the shorthand form caused pip to resolve and download sub-dependencies on all platforms, even though the top-level package was correctly marked as platform-specific in the lock file.  The shorthand keys are now translated to PEP 508 markers before building the pip requirement line, so the resolver skips resolution on non-matching platforms.
